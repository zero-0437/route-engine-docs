# Agent 规则审查流程 Checklist

> 当新增 Agent 或大幅修改路由规则时，按以下步骤审查，防止规则再次膨胀。

## 前置条件
- [ ] 新增 Agent 的白名单技能已确定
- [ ] 该 Agent 的定位/职责已与主 Agent 架构一致

## Step 1 — 规则必要性审查
- [ ] 每条规则是否有至少 3 条历史路由日志支撑？（无数据支撑的规则不开）
- [ ] 规则是否与已有 Agent 规则存在意图重叠？（`validate-route-map.py` 冗余度检查通过）
- [ ] 规则是否会截胡其他 Agent 的高置信度匹配？（检查负权重防护是否必要）
- [ ] 共享规则 (shared.yaml) 中是否已有覆盖该意图的通用规则？

## Step 2 — 规则质量审查
- [ ] 通配/泛用词（实现/测试/安全/设计/错误/文档等）不加模糊匹配
- [ ] 负权重规则只用于误匹配防护，不用于意图路由
- [ ] 单字符或过于宽泛的 pattern 不加正则
- [ ] 规则 weight 值在 0.5-1.0 之间，负权在 0 至 -1.0 之间

## Step 3 — 规则膨胀控制
- [ ] 新 Agent 首轮规则 ≤ 15 条（参考已瘦身 Agent 的均值）
- [ ] 每条规则必须有对应的 skills（空技能列表需标注理由）
- [ ] 使用 `validate-route-map.py` 运行全量审计通过
- [ ] 开启 fuzzy 的规则不超过该 Agent 规则总数的 50%

## Step 4 — 集成验证
- [ ] 新增规则后运行 `route_engine.py <典型输入>` 确认路由目标正确
- [ ] 运行全量 127 测试：`cd tests && python -m pytest -v`
- [ ] 与已存在的 route-map 集成后无冲突
