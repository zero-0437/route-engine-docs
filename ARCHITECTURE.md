## 三层路由决策

### auto — 高置信直接路由

当唯一 Agent 得分超过阈值（默认 `threshold: 0.5`）且第二名差距 > `margin: 0.3` 时，直接返回该 Agent。

```
programmer: 1.8  ← winner (超过 threshold)
pm-agent:  0.2  ← 差距 1.6 > 0.3
→ 返回 {'agent': 'programmer', 'mode': 'auto'}
```

### auto_tiebreak — 平局裁决

当多个 Agent 得分接近时（差距 ≤ margin），引擎不武断裁决，而是返回所有候选者，由上层（主 Agent）二次选择。

```
programmer: 1.2  ← 差距 0.2 ≤ 0.3
pm-agent:   1.0
→ 返回 {'candidates': ['programmer', 'pm-agent'], 'mode': 'auto_tiebreak'}
```

### unrouted — 无匹配零 Token 返回

当任何 Agent 得分都低于 `threshold` 时，引擎直接返回空结果，**不调用 LLM 兜底**。这是设计的核心——宁愿不路由，也不浪费 token 做低质量路由。

```
programmer: 0.1
pm-agent:   0.0
→ 返回 {'agent': None, 'mode': 'unrouted'}
```

## 路由引擎核心流程（代码级）

```
decide(user_input)
  │
  ├─ _normalize(user_input)               # 小写 + 去空白
  │
  ├─ load_route_map()                     # 加载 YAML（带缓存）
  │  ├─ index.yaml → agents 索引
  │  ├─ routes/*.yaml → 规则
  │  └─ chains/*.yaml → chain 定义
  │
  ├─ evaluate(user_input, route_map)      # 遍历所有 Agent 规则
  │  ├─ match_keyword(text, rule)         # 精确关键词哈希
  │  ├─ match_phrase(text, rule)          # 短语子串
  │  ├─ match_regex(text, rule)           # 正则
  │  └─ match_fuzzy(text, rule)           # 模糊重叠率
  │
  ├─ resolve_skills(result)               # 技能反向匹配
  │
  └─ 最终决策 + 日志                        # auto / auto_tiebreak / unrouted
```

## 与 Hermes Agent 的集成方式

### 配置集成

在 `config.yaml` 中指定路由模式：

```yaml
route_mode: zero-token
route_threshold: 0.5
route_margin: 0.3
```

### 规则热加载

引擎使用模块级缓存。当 route-map/ 下文件变更时，调用 `_clear_cache()` 即可强制重新加载：

```python
from route_engine import _clear_cache, decide
_clear_cache()  # 下次 decide() 会重新加载 YAML
```

### Chain 编排集成

路由决策结果包含 `chain` 字段时，主 Agent 读取该字段并调用 `chain_executor` 执行编排流程。

### 日志审计集成

所有路由决策自动写入 JSONL 格式日志，可用 `analyze-route-log.py` 脚本生成分析报告。
